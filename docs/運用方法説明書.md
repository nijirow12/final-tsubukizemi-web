## 1. サイトの全体構成

### 1.1 ページ一覧

| URL           | 内容                  | 更新頻度        |
| ------------- | ------------------- | ----------- |
| `/`           | ランディング（3D地球儀）       | ほぼ変更なし      |
| `/home`       | ホーム（地球儀+ナビ）         | ほぼ変更なし      |
| `/about`      | ゼミ紹介・教授プロフィール       | 年1回程度       |
| `/activities` | 活動写真・Instagram フィード | **随時**      |
| `/members`    | メンバー一覧              | **年度ごとに更新** |

### 1.2 更新方法の早見表

| 更新したい内容 | 更新方法 | 必要スキル |
|--------------|---------|----------|
| Instagram の新規投稿 | Instagram に投稿するだけ（`#国名` タグ必須） | なし |
| 地球儀ピンの写真追加 | Google Drive の国フォルダにアップロード | なし |
| 活動ページ（`/activities`） | Instagram 投稿と連動（自動反映） | なし |
| メンバー情報（CMS連携後） | microCMS 管理画面で編集 | なし |
| メンバー情報（現状） | `src/data/members-demo.ts` をコード編集 | ◎必要 |
| ゼミ紹介文（`/about`） | コード編集 + GitHub push | ◎必要 |
| デザイン変更 | コード編集 + GitHub push | ◎必要 |

> **画像ソースの使い分け（重要）**
> - **地球儀のピンに表示される写真** = Google Drive のフォルダに入れた画像
> - **`/activities` ページに表示される投稿** = Instagram の最新6投稿（Behold.so 経由）
>
> この2つは独立したシステムです。どちらも「国名で紐付け」しますが、紐付け方法が異なります（Drive はフォルダ名、Instagram はハッシュタグ）。

## 2. 日常的な更新作業

### 2.1 最も頻度の高い作業（運用担当者が自分でできる）

1. **Instagram 投稿** → Behold.so 経由で `/activities` に自動反映（§4参照）
2. **Google Drive への写真追加** → `/activities` に自動反映（§3参照）
3. **microCMS でメンバー情報編集** → `/members` に反映（§5参照）

これらはすべて **コード変更不要** です。

### 2.2 コード変更が必要な作業

- ゼミの紹介文や理念の書き換え
- デザインの調整
- 新しいページの追加
- 教授情報の更新

これらは開発者に依頼するか、§6 を参照してください。

> **コード保守体制について**
> 本プロジェクトは **「コード管理は開発者、コンテンツ管理はクライアント」** という役割分担で運用されています（[引き渡し手順書.md の「引き渡しパターンの選択」](引き渡し手順書.md) 参照）。
>
> 開発者は GitHub リポジトリの Collaborator として残っており、**ローカルから `git push` するだけで本番反映が可能** です。そのためコード変更は開発者に依頼するのが基本ルートです。クライアント側で直接コードを編集することも可能ですが、動作不良のリスクがあるため、§6 の内容を理解した上で実施してください。

## 3. 活動写真の追加（Google Drive）

### 3.1 仕組み

津吹ゼミが所有する Google Drive 内の「活動写真ルートフォルダ」に、**地域 → 国** の階層で写真を整理すると、サイトの **3D地球儀のピンに紐づいた写真パネル**（ランディングページ `/` とホームページ `/home`）に自動で取得・表示されます。

> **注意:** `/activities` ページに表示されるのは Instagram フィード（Behold.so 経由）であり、Google Drive の画像とは独立しています。詳細は §4 を参照。

### 3.2 フォルダ構造（厳守: 2階層限定）

サイト側は **ルート直下を「地域」、その配下を「国」** として2階層のみを探索します。3階層以上の入れ子や、ルート直下に国フォルダを置く構成は認識されません。

```
活動写真（ルートフォルダ）           ← GOOGLE_DRIVE_ROOT_FOLDER_ID で指定
├── Asia（地域: 任意の名前でOK）
│   ├── 🇰🇭カンボジア（国）           ← 国名を含むフォルダ名が必須
│   │   ├── 写真1.jpg
│   │   ├── 写真2.jpg
│   │   └── ...
│   ├── 🇻🇳ベトナム
│   ├── 🇹🇭タイ
│   └── ...
└── ...
```

### 3.3 国フォルダの命名規則

国フォルダ名には、以下の **日本語または英語の国名** を含める必要があります（絵文字や記号、前後の文字列は無視されます。大文字小文字も問いません）。

| 国 | 日本語 | 英語 |
|----|------|------|
| カンボジア | `カンボジア` | `cambodia` |
| ベトナム | `ベトナム` | `vietnam` |
| タイ | `タイ` | `thailand` |
| フィリピン | `フィリピン` | `philippines` |
| インドネシア | `インドネシア` | `indonesia` |
| マレーシア | `マレーシア` | `malaysia` |
| シンガポール | `シンガポール` | `singapore` |
| ミャンマー | `ミャンマー` | `myanmar` |
| ラオス | `ラオス` | `laos` |
| インド | `インド` | `india` |
| 日本 | `日本` | `japan` |

> ✅ 正しい例: `🇰🇭カンボジア`, `cambodia-2025`, `Cambodia Field Work`
> ❌ 誤った例: `プノンペン`（都市名のみ）, `アンコールワット`（地名のみ）

### 3.4 写真を追加する手順

1. Google Drive で「活動写真」ルートフォルダを開く
2. 該当する **地域 → 国** のフォルダへ移動
3. ファイルをドラッグ&ドロップでアップロード
4. **数分以内** に本番サイトへ反映されます（キャッシュ最大24時間）

### 3.5 推奨ファイル形式

- 形式: `.jpg`, `.jpeg`, `.png`, `.webp`, `.heic`
- 画像サイズ: 長辺 **2000px 以下**（配信時は自動で 600px JPEG にサムネイル化されますが、元画像が大きすぎると取得が遅くなります）
- 1国あたりの枚数: 20〜50枚程度が目安
- HEIC（iPhone標準形式）もそのままアップロードでOK。サーバー側で JPEG 化されます

### 3.6 新しい国を追加したい場合

1. 対応する地域フォルダ内に、国名のフォルダを作成
   - フォルダ名例: `🇮🇳インド`, `🇲🇲ミャンマー`
2. 地球儀のピンも追加する必要があります → **開発者に依頼**（[src/data/pin-coordinates.ts](../src/data/pin-coordinates.ts) を編集）

### 3.7 注意事項

- フォルダ名を変更する際、**国名キーワード**（§3.3 参照）は必ず残してください
- ルートフォルダに共有されている **サービスアカウントの閲覧者権限を絶対に解除しない** でください（解除すると Drive API が全エラーになります）
- 誤って削除した場合は Google Drive のゴミ箱から復元可能です

## 4. Instagram 投稿からの自動反映（Behold.so）

### 4.1 仕組み

公式 Instagram (`@emcglobalzemi2025`) に投稿すると、**Behold.so** 経由で自動的にサイトの `/activities` に最新6投稿が表示されます。

### 4.2 投稿時のポイント

#### 国別タグ付け（重要）

投稿キャプションに **国名のハッシュタグ** を入れると、サイト上で国別に分類されます。

**推奨タグ（いずれか片方でOK）:**

| 国 | 日本語タグ | 英語タグ |
|----|---------|---------|
| カンボジア | `#カンボジア` | `#cambodia` |
| ベトナム | `#ベトナム` | `#vietnam` |
| タイ | `#タイ` | `#thailand` |
| フィリピン | `#フィリピン` | `#philippines` |
| インドネシア | `#インドネシア` | `#indonesia` |
| マレーシア | `#マレーシア` | `#malaysia` |
| シンガポール | `#シンガポール` | `#singapore` |
| ミャンマー | `#ミャンマー` | `#myanmar` |
| ラオス | `#ラオス` | `#laos` |
| インド | `#インド` | `#india` |
| 日本 | `#日本` | `#japan` |

### 4.3 反映タイミング

- 投稿から約 **5〜15分** で Behold.so 側が取得
- その後、サイト側の次回アクセス時に表示

### 4.4 無料プランの制約

| 項目 | 制限 |
|------|------|
| 表示投稿数 | **6投稿** まで |
| 月間ビュー数 | **1,200** まで |
| 動画表示 | サムネイルのみ |

古い投稿は自動的に落ちていきます。制限を超えると有料プラン（月$5〜）への移行を検討してください。

### 4.5 アカウント要件

- Instagram は **ビジネスアカウント** か **クリエイターアカウント** である必要があります
- 公開アカウントでなければなりません（非公開アカウントは取得不可）

## 5. メンバー情報の更新（microCMS）

### 5.1 microCMS 連携の状態

コードは microCMS の拡張スキーマ（日英併記・訪問国・自己紹介など全フィールド）に対応済みです。

- **microCMS 環境変数が設定済みの場合**: microCMS から自動取得されます
- **環境変数が未設定の場合**: デモデータ（[src/data/members-demo.ts](../src/data/members-demo.ts)）が表示されます

microCMS に切り替えるには、Vercel の環境変数に `MICROCMS_SERVICE_DOMAIN` と `MICROCMS_API_KEY` を設定するだけです（コード変更は不要）。

### 5.2 microCMS 管理画面へのログイン

1. `https://【サービスドメイン】.microcms.io/` にアクセス
2. 発行済みのメール・パスワードでログイン

### 5.3 メンバー追加

1. 左メニューから **members** を選択
2. 右上の **「追加」** をクリック
3. 各フィールドを入力

#### microCMS スキーマ（必須）

コードはこのスキーマに準拠しています。

| フィールド            | 型          | 例                         | 備考                 |
| ---------------- | ---------- | ------------------------- | ------------------ |
| `name`           | テキスト       | 山田 太郎                     | 日本語氏名（必須）          |
| `nameEn`         | テキスト       | Taro Yamada               | 英語氏名（必須）           |
| `introduction`   | テキストエリア    | はじめまして、経済学部4年の...         | 日本語自己紹介            |
| `introductionEn` | テキストエリア    | Hi, I am a 4th-year...    | 英語自己紹介             |
| `projects`       | テキストエリア    | カンボジアで教育支援プロジェクト...       | 日本語活動内容            |
| `projectsEn`     | テキストエリア    | Currently involved in...  | 英語活動内容             |
| `countries`      | 複数行テキストリスト | カンボジア, タイ                 | 訪問国（カンマ区切り or リスト） |
| `photo`          | 画像         | 画像アップロード                  | 正方形推奨              |
| `isOB`           | 真偽値        | ✔ / 空欄                    | 卒業生フラグ             |
| `grade`          | テキスト       | 3年                        | 現役のみ（OBは空欄）        |
| `socialUrl`      | テキスト       | https://instagram.com/... | SNSリンク（任意）         |

4. **「公開」** ボタンを押すと即時反映

### 5.4 メンバー編集・削除

- 編集: 一覧から該当メンバーをクリック → 修正 → 公開
- 削除: 編集画面右上の **「削除」** ボタン

### 5.5 下書き保存

公開前に下書きしたい場合は、**「下書き保存」** を選択。公開するまでサイトには反映されません。

## 6. テキスト・画像の直接編集（コード変更）

コード変更はリスクがあるため、**可能な限り開発者に依頼**してください。どうしても自分で変更する必要がある場合のみ、以下を参照してください。

### 6.1 主なファイル

| 変更したい内容 | 編集ファイル |
|--------------|------------|
| ゼミ紹介文・理念 | `src/app/about/page.tsx` |
| 教授プロフィール | `src/data/professor.ts` |
| ナビゲーションメニュー | `src/components/common/Header.tsx` |
| フッターの著作権表示 | `src/components/common/Footer.tsx` |
| 地球儀のピン位置・国 | `src/data/pin-coordinates.ts` |
| メンバー（デモデータ） | `src/data/members-demo.ts` |
| 教授の写真 | `public/images/tubukitatuya.jpg` を差し替え |
| About の集合写真 | `public/images/tsubukizemiabout.webp` を差し替え |

### 6.2 編集 → 公開フロー

1. GitHub 上で該当ファイルを開く
2. 右上の鉛筆アイコンで直接編集
3. 画面下部の **「Commit changes」** をクリック
4. 数分後、Vercel が自動でビルド・公開

または、ローカルで編集 → git push（§8, 引き渡し手順書）

### 6.3 変更前の必須作業

- **変更前に Vercel ダッシュボードで現在のデプロイ状況を確認**
- 大幅な変更は **Preview デプロイ** で確認してから master に merge
- 不安な場合は必ず開発者に相談

## 7. サイトの状態を確認する

### 7.1 Vercel ダッシュボード

`https://vercel.com/` にログイン → プロジェクトを開く

**確認できること:**
- デプロイ履歴（いつ・誰が・何を変更したか）
- エラーログ（`Logs` タブ）
- アクセス数（`Analytics` タブ）
- 環境変数（`Settings → Environment Variables`）

### 7.2 デプロイが失敗したとき

1. Vercel ダッシュボードの **Deployments** を開く
2. 赤い **Failed** 表示のデプロイをクリック
3. ビルドログを確認
4. 前回成功したデプロイの右側の **「...」 → Promote to Production** で一時的に戻せます
5. 原因が分からなければ開発者に連絡

### 7.3 ロールバック（過去のバージョンに戻す）

1. `Deployments` から戻したいバージョンを選択
2. **「Promote to Production」** をクリック
3. 即座に過去のバージョンが本番反映されます

## 9. トラブルシューティング

### 9.1 「地球儀が表示されない」

**原因候補:**
- ブラウザが WebGL に非対応（古い端末）
- GPU アクセラレーションが無効

**対処:**
- 別のブラウザで試す（Chrome 推奨）
- ブラウザ設定でハードウェアアクセラレーションを有効化
- それでもダメなら開発者に連絡

### 9.2 「活動写真（Google Drive）が表示されない」

**確認項目:**
1. Vercel の環境変数 `GOOGLE_DRIVE_ROOT_FOLDER_ID` が正しいか
2. サービスアカウント JSON が正しく登録されているか
3. ルートフォルダにサービスアカウントが **閲覧者権限で共有** されているか
4. Google Cloud Console で Drive API が有効か

**対処:**
- 上記4点をすべて確認
- それでもダメなら開発者に連絡

### 9.3 「Instagram フィードが表示されない」

**確認項目:**
1. `@emcglobalzemi2025` が公開ビジネスアカウントか
2. Behold.so の月間ビュー数超過していないか
3. Vercel の `NEXT_PUBLIC_BEHOLD_FEED_ID` が正しいか
4. Behold.so ダッシュボードでフィードが有効か

**対処:**
- Behold.so にログインしてフィードの状態を確認
- 必要なら Feed ID を再発行して Vercel を更新

### 9.4 「microCMS のメンバーがサイトに反映されない」

**確認項目:**
1. microCMS 側でメンバーを「公開」にしているか（下書きになっていないか）
2. APIキー `MICROCMS_API_KEY` が正しいか
3. API 連携が有効化されているか（§5.1 参照）

### 9.5 「デプロイ後にサイトが真っ白になった」

**対処:**
1. Vercel ダッシュボードですぐに **前のデプロイにロールバック**（§7.3）
2. 原因調査は復旧後に
3. 開発者に連絡

### 9.6 「環境変数を変えても反映されない」

環境変数を変更した後は、必ず **Redeploy** が必要です。

1. Vercel ダッシュボード → Deployments
2. 最新デプロイの **「...」 → Redeploy**

## 10. やってはいけないこと

運用上、以下の行為はサイトの動作を致命的に壊す可能性があります。**絶対に避けてください。**

### 10.1 サービス側

- ❌ Google Drive のルートフォルダの **サービスアカウント共有を解除** する
- ❌ Google Drive のフォルダ構造（地域→国）を変更する
- ❌ Google Cloud プロジェクトを削除する
- ❌ Vercel プロジェクトを削除する
- ❌ GitHub リポジトリを削除する
- ❌ microCMS のサービスを削除する
- ❌ Instagram アカウントを非公開にする

### 10.2 認証情報

- ❌ API キーやサービスアカウント JSON を SNS・メール・チャットで平文送信
- ❌ 環境変数を `.env` に書いて Git コミット
- ❌ 環境変数の `NEXT_PUBLIC_` プレフィックスを他の Secret に付ける
- ❌ Google サービスアカウントの JSON ファイルを公開場所に保存

### 10.3 コード変更

- ❌ `master` ブランチに動作確認なしで大量変更を push する
- ❌ `package.json` の依存を手動で削除する
- ❌ `next.config.ts` を理解せずに変更する
- ❌ Vercel の Framework 設定を変更する
- ❌ Node.js バージョンを勝手に下げる

### 10.4 運用体制

- ❌ 引き継ぎ時にパスワードマネージャなしで口頭のみの引き継ぎ
- ❌ 単一の個人アカウントで全サービスを運用（卒業・退任時に詰みます）
- ❌ 2要素認証を無効化する
