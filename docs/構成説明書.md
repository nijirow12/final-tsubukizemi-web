## 1. プロジェクト概要

津吹ゼミ（法政大学）の公式 Web サイト。アジア圏フィールドワークの活動記録・メンバー紹介・インスタグラムフィードを掲載する。

**主な特徴:**
- インタラクティブな 3D 地球儀（Three.js）でフィールドワーク渡航国を可視化
- microCMS からメンバー情報を動的取得
- Google Drive に保存した現地写真を API 経由で配信
- Instagram 投稿を Behold 経由で自動表示
- 日本語 / 英語の切替（全ページ対応）

## 4. ページ構成とルーティング

### ルートレイアウト (`src/app/layout.tsx`)

```
HTML (lang="ja")
└── LanguageProvider       ← 言語状態をアプリ全体に提供
    └── ConditionalLayout  ← パスに応じてヘッダー/フッターを出し分け
```

### レイアウト分岐ルール (`ConditionalLayout.tsx`)

| パス | ヘッダー | フッター | メイン幅 |
|---|---|---|---|
| `/` | なし | なし | フルスクリーン |
| `/home` | あり | なし | フルスクリーン |
| `/about`, `/activities`, `/members`, `/contact` | あり | あり | 最大幅付きコンテナ |
| その他 | あり | あり | センタリング |

### 各ページ詳細

#### `/` — ランディングページ
- **コンポーネント:** `GlobeScene`
- **内容:** インタラクティブ 3D 地球儀のみ（ヘッダー/フッターなし）
- **機能:** 国ピンをクリックで現地写真ギャラリーをオーバーレイ表示、ドラッグ・ズーム操作、アイドル時自動回転

#### `/home` — ホームページ
- **コンポーネント:** `GlobeSceneHome`
- **内容:** 3D 地球儀 + 周囲にポラロイド風写真フレーム（PC: 12 枚、モバイル: 8 枚）
- **機能:** Google Drive から写真をプログレッシブロード

#### `/about` — アバウトページ
- **言語対応:** あり（ja/en）
- **内容:**
  - ヒーローセクション（キャッチコピー + 渡航国数）
  - グループ写真（`tsubukizemiabout.webp`）
  - ミッションステートメント
  - 教授プロフィール（写真・研究分野タグ・バイオ）
- **データソース:** `src/data/professor.ts`（静的）

#### `/activities` — 活動ページ
- **言語対応:** あり（ja/en）
- **内容:**
  - ヒーローセクション
  - Instagram フィード（最新 2 件）
- **データソース:** Behold → Instagram

#### `/members` — メンバーページ
- **言語対応:** あり（ja/en）
- **内容:**
  - 教授セクション
  - 現役メンバーセクション（学年表示）
  - OB/OG セクション
- **メンバーカード機能:**
  - 写真 / アバター
  - 名前（日英）
  - 自己紹介（折りたたみ可）
  - 研究テーマ・プロジェクト
  - 渡航国（emoji フラグ付き）
  - SNS リンク
- **データソース:** `/api/members` → microCMS（フォールバック: `members-demo.ts`）

#### `/contact` — お問い合わせページ
- **言語対応:** あり（ja/en）
- **内容:** 名前・メール・メッセージフォーム
- **現状:** 送信機能は未実装（UI のみ）

## 5. APIルート

すべて `src/app/api/` 配下の Next.js Route Handlers（Edge ではなく Node.js ランタイム）。

### `GET /api/members`

```
microCMS API
    → メンバー一覧取得
    → フィールド名正規化（nameJP→name, nameEN→nameEn, など）
    → grade=="alumni" の場合 isOB=true にマーク
    → JSON レスポンス

microCMS が未設定の場合
    → members-demo.ts のデモデータをフォールバック返却
```

**レスポンス型:**
```ts
Member {
  id: string
  name: string          // 日本語名
  nameEn: string        // 英語名
  introduction: string
  introductionEn: string
  projects: string
  projectsEn: string
  countries: string[]   // 渡航国 ID 配列
  photo?: { url: string }
  isOB: boolean
  grade?: string
  socialUrl?: string
}
```

---

### `GET /api/drive/folders`

```
Google Drive API（サービスアカウント認証）
    → ルートフォルダ直下のサブフォルダ一覧
    → 各サブフォルダ内のフォルダ一覧（国フォルダ）
    → { folderName: folderId } のマップを返却

キャッシュ: max-age=3600, stale-while-revalidate=86400
```

---

### `GET /api/drive/images?folderId={id}`

```
Google Drive API
    → 指定フォルダ内の画像ファイル一覧（最大 50 件）
    → 作成日順（新しい順）
    → [{ id, name, url }] を返却

キャッシュ: max-age=3600, stale-while-revalidate=86400
```

---

### `GET /api/drive/image?fileId={id}`

```
Google Drive API
    → サムネイル URL を取得（幅 400px、HEIC → JPEG 変換）
    → 画像バイナリを直接配信（Content-Type: image/jpeg）
    → サムネイル取得失敗時は元ファイルにフォールバック

キャッシュ: max-age=31536000, immutable（1 年）
```

---

### `GET /api/drive/batch-images?fileIds={id1,id2,...}`

```
最大 20 ファイル ID を受け取り
    → 並列で各画像を取得
    → base64 Data URI に変換
    → { images: { fileId: "data:image/jpeg;base64,..." } } を返却

キャッシュ: max-age=86400（1 日）
```

## 7. 外部サービス連携

### microCMS（メンバーデータ）

```
microCMS
 └── /api/v1/members
      ├── nameJP, nameEN
      ├── introductionJP, introductionEN
      ├── projectsJP, projectsEN
      ├── countries (文字列、カンマ区切り)
      ├── photo (画像オブジェクト)
      ├── grade (学年 or "alumni")
      └── socialUrl
```

認証: `MICROCMS_SERVICE_DOMAIN` + `MICROCMS_API_KEY`  
フォールバック: 環境変数未設定時は `src/data/members-demo.ts` を使用

---

### Google Drive（写真配信）

```
Google Drive
 └── ルートフォルダ (GOOGLE_DRIVE_ROOT_FOLDER_ID)
      ├── 地域フォルダ（例: 東南アジア）
      │    ├── カンボジア/
      │    ├── ベトナム/
      │    └── ...
      └── その他の地域...
```

認証: サービスアカウント JSON（`GOOGLE_SERVICE_ACCOUNT_JSON` 環境変数 or ローカルファイル）  
画像配信フロー: `Drive API → サムネイル取得 → Next.js API → ブラウザ`

---

### Behold（Instagram フィード）

```
Instagram
 └── Behold サービス（NEXT_PUBLIC_BEHOLD_FEED_ID）
      └── /api/v1/feed-sessions/{id}
           → 投稿リスト（最新 2 件表示）
           → カルーセル / 単体投稿の画像を抽出
           → ハッシュタグから渡航国を特定
```

## 8. データフロー

### メンバーページのデータフロー

```
ブラウザ (members/page.tsx)
  → fetch("/api/members")
      → microcms.ts: getMembers()
          → microCMS API（成功時）
          → members-demo.ts（フォールバック）
      → フィールド正規化
      → JSON レスポンス
  → React state 更新
  → MemberCard コンポーネントに渡す
```

### 地球儀（ランディング）の画像ロードフロー

```
ユーザーがピンをクリック
  → fetch("/api/drive/images?folderId=XXX")
      → Google Drive API: ファイル一覧
      → [{ id, name }] を返す
  → ファイル ID を 20 件ずつバッチ
  → fetch("/api/drive/batch-images?fileIds=...")
      → 並列で各画像を取得・base64 変換
      → { images: { id: dataURI } } を返す
  → React state に追加（プログレッシブ表示）
```

## 10. 多言語対応

### 仕組み

```
LanguageProvider (src/lib/language-context.tsx)
  → React Context で lang ("ja" | "en") を管理
  → デフォルト: "ja"
  → Header の言語トグルボタンで setLang() を呼び出す
  → useLanguage() フックで各コンポーネントが取得
```

### 使用例（ページ内）

```tsx
const { lang } = useLanguage();
const title = lang === 'ja' ? '津吹ゼミについて' : 'About Tsubuki Seminar';
```

### 対応範囲

| 場所 | 対応 |
|---|---|
| ヘッダー / フッター | あり |
| About ページ | あり |
| Activities ページ | あり |
| Members ページ | あり（メンバーカード含む） |
| Contact ページ | あり |
| ランディング / Home | テキスト最小のため実質不要 |

## 11. 環境変数一覧

`.env.local` に設定。Vercel ダッシュボードでも同じキーを設定する。

| 変数名 | 用途 | 公開範囲 |
|---|---|---|
| `MICROCMS_SERVICE_DOMAIN` | microCMS サービスドメイン | サーバーのみ |

> `NEXT_PUBLIC_` プレフィックスがある変数はブラウザに公開されるため、シークレット情報は入れないこと。
