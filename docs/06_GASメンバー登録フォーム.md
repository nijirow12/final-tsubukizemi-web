# Google フォーム メンバー登録

**最終更新:** 2026-04-22
**位置づけ:** ゼミメンバー情報を Google フォームで集める運用のガイド。スプレッドシートへの保存は **Google フォームの純正機能（「回答」→ スプレッドシート連携）** で行い、GAS は **回答項目を構造化する収集スクリプト** に徹する。外部連携（microCMS / Slack など）を後から足したくなったら、この GAS の拡張ポイントに処理を追記する。

## 1. 概要

```
[ゼミ生]
   │ 回答送信
   ▼
[Google フォーム] ──(写真は Drive に自動保存)──▶ [Google Drive]
   │ │
   │ │  (A) 回答タブの "スプレッドシート連携" で回答を自動追記
   │ └────────────▶ [Google スプレッドシート]
   │
   │  (B) onFormSubmit トリガー
   ▼
[GAS: member-form.gs]
   │ 回答を構造化オブジェクトに整形 → 実行ログに記録
   │ (※将来の拡張ポイント: microCMS / Slack / 別 API 等)
   ▼
[Apps Script 実行ログ]
```

- **(A) 保管** は Google フォームの標準機能でカバー（管理者が設定）
- **(B) 構造化** は GAS がカバー（回答を `{ nameJP, nameEN, countries: [...], photo: [...] }` のような形で整形）
- 新規回答の通知が欲しい場合は、Google フォームの「回答」タブ → 「⋮」→ **新しい回答のメール通知を受け取る** を ON にする

## 2. フォーム設計

### 2.1 フォーム名

> **津吹ゼミ メンバー登録フォーム**

### 2.2 フォーム説明（冒頭に表示する文）

> 津吹ゼミ公式サイトに掲載するメンバー情報を登録するフォームです。日本語と英語の両方を入力してください（英語は簡単な翻訳でも構いません）。送信内容はサイトのメンバーページに掲載されます。
>
> - **所要時間の目安:** 約 10 分
> - **写真:** 正方形に近いサイズ（1:1 推奨）、1MB 以下推奨
> - **SNS:** Instagram / LinkedIn / X など任意の 1 つをリンク可（省略可）
>
> 修正が必要な場合はサイト管理者までご連絡ください。

### 2.3 質問項目

| # | 質問タイトル | 回答形式 | 必須 | GAS 出力キー |
|---|---|---|:---:|---|
| 1 | 名前（日本語） | 記述式（短文） | ✅ | `nameJP` |
| 2 | Name (English) | 記述式（短文） | ✅ | `nameEN` |
| 3 | 自己紹介（日本語） | 記述式（長文） | ✅ | `introductionJP` |
| 4 | Self-introduction (English) | 記述式（長文） | ✅ | `introductionEN` |
| 5 | 携わったプロジェクト（日本語） | 記述式（長文） |  | `projectsJP` |
| 6 | Projects (English) | 記述式（長文） |  | `projectsEN` |
| 7 | 活動国 | 記述式（短文 / 自由記述） | ✅ | `countries`（配列化） |
| 8 | 学年 | ラジオボタン（単一選択） | ✅ | `grade` |
| 9 | SNS URL | 記述式（短文） |  | `socialurl` |
| 10 | プロフィール写真 | ファイルのアップロード（1 枚） |  | `photo`（Drive URL 配列） |

> **重要:** 質問タイトルを変更すると GAS の `FIELD_MAP` のキーと一致しなくなり、該当項目が構造化オブジェクトに入らなくなる。タイトルを変える場合は [docs/gas/member-form.gs](./gas/member-form.gs) の `FIELD_MAP` も同じ文字列に揃えること（カッコの全角/半角・スペース含め完全一致が必要）。

### 2.4 入力フォーマットの補足

**活動国（Q7 / 自由記述）:**

- 複数国ある場合はカンマ `,` / 読点 `、` / スラッシュ `/` / 改行 のいずれかで区切る
- 日本語 / 英語どちらで書いてもよい（GAS はそのまま保持する）
- 例: `カンボジア, ベトナム` / `Cambodia / Vietnam` / `タイ、Thailand`

**学年（Q8）:** `1年` / `2年` / `3年` / `4年` / `5年` / `6年` / `7年` / `alumni（OB・OG）`

## 3. GAS コード

**➡ 本体ソース: [docs/gas/member-form.gs](./gas/member-form.gs)**

**貼り付け手順:**

1. フォーム編集画面 → 右上「⋮」→ スクリプトエディタ
2. [docs/gas/member-form.gs](./gas/member-form.gs) を開いて全選択 → コピー
3. GAS 側の `Code.gs` に貼り付け → 保存

> **なぜ別ファイル？** markdown のコードフェンス ` ```javascript ` / ` ``` ` を一緒にコピーすると、GAS がそれをテンプレートリテラルとして解釈し `SyntaxError: Unexpected end of input` になる事故があるため、生の `.gs` 単体ファイルを唯一のソースとしている。

**スクリプト内の関数:**

| 関数 | 役割 |
|---|---|
| `onFormSubmit(e)` | フォーム送信時のトリガー。`collectFormItems` で整形 → `Logger.log` に書き出す |
| `collectFormItems(response)` | 回答を「`FIELD_MAP` のキー → 値」のオブジェクトに整形（写真 → URL 配列、活動国 → 文字列配列） |
| `splitList(value)` | 自由記述を区切り文字で分割して配列化 |
| `toPhotoLinks(value)` | 写真 file ID を Drive URL に変換 |

**出力される構造化オブジェクトの例:**

```json
{
  "timestamp": "2026-04-22T05:23:45.000Z",
  "nameJP": "山田 太郎",
  "nameEN": "Taro Yamada",
  "introductionJP": "...",
  "introductionEN": "...",
  "projectsJP": "...",
  "projectsEN": "...",
  "countries": ["カンボジア", "ベトナム"],
  "grade": "3年",
  "socialurl": "https://instagram.com/...",
  "photo": ["https://drive.google.com/file/d/xxx/view"]
}
```

## 4. セットアップ手順

### 4.1 前提

- Google アカウント（フォーム / スプレッドシート / Drive を扱える）

### 4.2 Google フォームを作る

1. https://forms.google.com/ で新規フォーム作成
2. §2 の仕様どおりにフォーム名・説明・質問を作成
3. Q10 は「ファイルのアップロード」タイプ（Google ログイン必須 / 1 ファイル上限 / 画像のみ）

### 4.3 スプレッドシート連携を設定（フォーム純正機能）

1. フォーム編集画面 → **回答** タブ
2. 右上の緑色のスプレッドシートアイコンをクリック
3. 「新しいスプレッドシートを作成」または既存シートを選択 → **作成 / 選択**
4. 以降、フォーム送信のたびにそのスプレッドシートに 1 行自動追記される（GAS 不要）

### 4.4 GAS を設定

1. フォーム編集画面 → 右上「⋮」→ スクリプトエディタ
2. `Code.gs` に [docs/gas/member-form.gs](./gas/member-form.gs) の内容を貼り付け → 保存
3. **トリガーを登録**:
   - 左メニュー **⏰ トリガー** → **＋ トリガーを追加**
   - 実行する関数: `onFormSubmit`
   - イベントのソース: **フォームから**
   - イベントの種類: **フォーム送信時**
   - 保存 → 初回は権限承認ダイアログが出るので承認

### 4.5 動作確認

1. フォームをプレビューから 1 件テスト送信
2. スクリプトエディタ → **実行数**（履歴）で `onFormSubmit` が成功していることを確認
3. 実行数の該当行 → **詳細を表示** で `Logger.log` に構造化オブジェクトが JSON で記録されていることを確認
4. 同時に §4.3 で連携したスプレッドシートにも回答が 1 行追加されていることを確認
5. テストデータは両方から削除

## 5. 拡張方法

将来的に「回答をどこかへ送信したい」となった場合は、[docs/gas/member-form.gs](./gas/member-form.gs) の `onFormSubmit` 内の **拡張ポイント** コメントのすぐ下に処理を足す。

```javascript
function onFormSubmit(e) {
  const data = collectFormItems(e.response);
  Logger.log(...);

  // --- 拡張ポイント ---
  // ここに追記:
  //   postToMicrocms(data);       // microCMS に POST
  //   notifySlack(data);          // Slack Webhook で通知
  //   UrlFetchApp.fetch(...)      // 任意の外部 API
}
```

`data` は §3 の出力構造を持つオブジェクト（`nameJP`, `countries: [...]`, `photo: [...]` 等）。この構造をベースに外部連携を組み立てる。

## 6. トラブルシューティング

| 症状 | 原因と対処 |
|---|---|
| `onFormSubmit` 実行時に `Cannot read property 'response' of undefined` | トリガーが「スプレッドシートから」になっている可能性。**フォームから / フォーム送信時** に設定し直す（§4.4）。 |
| 実行ログで `nameJP` 等一部フィールドが空 | フォームの質問タイトルと `FIELD_MAP` のキー文字列が不一致。カッコ（全角 `（` / 半角 `(`）や空白の差異に注意。完全一致が必要。 |
| 写真フィールドが `[]`（空配列） | Q10 に添付がないか、GAS 実行ユーザーが該当 Drive ファイルにアクセスできていない。通常は同じ Google アカウントでフォーム所有・スクリプト所有していれば発生しない。 |
| フォーム送信時にエラーメールが飛んでこない | Apps Script のトリガー失敗通知は既定でスクリプト所有者に送られる。アカウントの通知設定や迷惑メール振り分けを確認。 |
