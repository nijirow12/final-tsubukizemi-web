# Google フォーム + GAS メンバー登録（スプレッドシート格納）

**最終更新:** 2026-04-22
**位置づけ:** ゼミメンバーを Google フォームで集め、GAS で Google スプレッドシートに 1 行追記する運用の設計・実装ガイド。

> **将来的な拡張:** 集まったスプレッドシートを microCMS に取り込む / 別ツールに流すといった拡張は後から追加できる。まずは純粋なフォーム → スプレッドシートのパイプラインだけを組む。

## 1. 概要

**データフロー:**

```
[ゼミ生]
   │ 回答送信
   ▼
[Google フォーム] ──（写真は Drive に自動保存）──▶ [Google Drive]
   │                                                    │
   │ onFormSubmit トリガー                       Drive URL として参照
   ▼                                                    │
[GAS: member-form.gs] ◀──────────────────────────────── ┘
   │ 1 行追記
   ▼
[Google スプレッドシート]
   （管理者が目視確認してメンバーページに反映したい場合はここから手作業）
```

**狙い:**

- ゼミ生本人が自己紹介やプロジェクト情報を入力できる（管理者がタイピングで代行しなくて良い）
- 日本語 / 英語の両方を併記入力させて二言語コンテンツを担保
- 写真は Google フォームが自動で Drive に保存、GAS はその URL だけスプレッドシートに残す
- **通知は設定不要。** 新規回答の確認は Google フォームの既定機能（回答タブ → 「新しい回答のメール通知」）でも、このスプレッドシートを直接開いてもよい

## 2. フォーム設計

### 2.1 フォーム名

> **津吹ゼミ メンバー登録フォーム**

### 2.2 フォーム説明（冒頭に表示する文）

> 津吹ゼミ公式サイトに掲載するメンバー情報を登録するフォームです。日本語と英語の両方を入力してください（英語は簡単な翻訳でも構いません）。送信内容はサイトのメンバーページに掲載されます。
>
> - **所要時間の目安:** 約 10 分
> - **写真:** 正方形に近いサイズ（1:1 推奨）、1MB 以下推奨
> - **SNS:** Instagram / LinkedIn / X など任意の 1 つをリンク可（省略可）
>
> 修正が必要な場合はサイト管理者までご連絡ください。

### 2.3 質問項目

| # | 質問 | 回答形式 | 必須 |
|---|---|---|:---:|
| 1 | 名前（日本語） | 記述式（短文） | ✅ |
| 2 | Name (English) | 記述式（短文） | ✅ |
| 3 | 自己紹介（日本語） | 記述式（長文） | ✅ |
| 4 | Self-introduction (English) | 記述式（長文） | ✅ |
| 5 | 携わったプロジェクト（日本語） | 記述式（長文） |  |
| 6 | Projects (English) | 記述式（長文） |  |
| 7 | 活動国 | 記述式（短文 / 自由記述） | ✅ |
| 8 | 学年 | ラジオボタン（単一選択） | ✅ |
| 9 | SNS URL | 記述式（短文） |  |
| 10 | プロフィール写真 | ファイルのアップロード（1 枚） |  |

> **重要:** 質問タイトルを変更すると GAS の `HEADERS` 配列と一致しなくなり、該当列に値が入らなくなる。タイトルを変える場合は [docs/gas/member-form.gs](./gas/member-form.gs) の `HEADERS` も同じ文字列に合わせて更新すること。

### 2.4 入力フォーマットの補足

**活動国（Q7 / 自由記述）:**

- 複数国ある場合はカンマや読点で区切って入力（例: `カンボジア, ベトナム` / `Cambodia / Vietnam`）
- 日本語 / 英語どちらで書いてもよい（スプレッドシートには**入力された文字列をそのまま**保存する）

**学年（Q8）:** `1年` / `2年` / `3年` / `4年` / `5年` / `6年` / `7年` / `alumni（OB・OG）`

## 3. スプレッドシート設計

GAS が自動で 1 行目にヘッダ、2 行目以降にデータを追記する。列は以下の順。

| 列 | 内容 | 例 |
|---|---|---|
| `timestamp` | 送信時刻（Apps Script が付与） | `2026-04-22 14:23:45` |
| `名前（日本語）` | Q1 の回答 | `山田 太郎` |
| `Name (English)` | Q2 の回答 | `Taro Yamada` |
| `自己紹介（日本語）` | Q3 の回答 | `経済学部4年...` |
| `Self-introduction (English)` | Q4 の回答 | `4th year...` |
| `携わったプロジェクト（日本語）` | Q5 の回答 | `...` |
| `Projects (English)` | Q6 の回答 | `...` |
| `活動国` | Q7 の回答（入力そのまま） | `カンボジア, ベトナム` |
| `学年` | Q8 の回答 | `3年` / `alumni` |
| `SNS URL` | Q9 の回答 | `https://instagram.com/...` |
| `プロフィール写真` | Q10 の Drive URL（複数ある場合はカンマ区切り） | `https://drive.google.com/file/d/xxx/view` |

1 行目は自動的に凍結表示される（`setFrozenRows(1)`）。

## 4. GAS コード

**➡ 本体ソース: [docs/gas/member-form.gs](./gas/member-form.gs)**

**貼り付け手順:**

1. フォーム編集画面 → 右上「⋮」→ スクリプトエディタ
2. [docs/gas/member-form.gs](./gas/member-form.gs) を開いて全選択 → コピー
3. GAS 側の `Code.gs` に貼り付け → 保存

> **なぜ別ファイル？** markdown のコードフェンス ` ```javascript ` / ` ``` ` を一緒にコピーすると、GAS がそれをテンプレートリテラルとして解釈し `SyntaxError: Unexpected end of input` になる事故があるため、生の `.gs` 単体ファイルを唯一のソースとしている。

**スクリプトの挙動サマリ:**

- `onFormSubmit(e)` — フォーム送信をトリガに、スプレッドシートに 1 行追記
- `HEADERS` — 列順を定義（ヘッダがまだ無ければ 1 行目に自動書き込み）
- `toPhotoLinks(ids)` — 写真ファイル ID を Drive URL に変換
- `testSheetAccess()` — 手動実行用。シートにテスト行を書き込めるかを確認

## 5. セットアップ手順

### 5.1 前提

- **Google アカウント** — フォーム / スプレッドシート / Drive すべてを扱える権限（ゼミ共用アカウント推奨）

### 5.2 Google フォームを作る

1. https://forms.google.com/ で新規フォーム作成
2. §2 の仕様どおりにフォーム名・説明・質問を作成
3. Q10 は「ファイルのアップロード」タイプ（Google ログイン必須 / 1 ファイル上限 / 画像のみ）
4. 任意: 「回答」タブで **新しい回答のメール通知を受け取る** を ON（新規回答がくるたびメールが届く）

### 5.3 格納先スプレッドシートを作る

1. https://docs.google.com/spreadsheets/ で新規スプレッドシートを作成（名前は任意: 例「津吹ゼミ メンバー回答」）
2. URL から ID をコピー
   ```
   https://docs.google.com/spreadsheets/d/【この部分がID】/edit
   ```

### 5.4 GAS を設定

1. フォーム編集画面 → 右上「⋮」→「スクリプトエディタ」
2. `Code.gs` に [docs/gas/member-form.gs](./gas/member-form.gs) の内容を貼り付け → 保存
3. 左メニュー **歯車（プロジェクトの設定）** → **スクリプト プロパティ** に以下を登録:

| キー | 値 |
|---|---|
| `SPREADSHEET_ID` | §5.3 で作ったスプレッドシートの ID |
| `SHEET_NAME` | （省略可） シート名。未設定なら `Members` が自動作成される |

4. 関数選択プルダウンで `testSheetAccess` を選び **実行** → 初回は権限承認ダイアログが出るので承認
5. スプレッドシートを開き、`Members` シートにテスト行（`(test) ...` を含む）が追加されていることを確認
6. 確認できたら、そのテスト行は手で削除してよい

### 5.5 トリガーを登録

1. スクリプトエディタ左メニュー **⏰ トリガー** → **＋ トリガーを追加**
2. 設定:
   - 実行する関数: `onFormSubmit`
   - イベントのソース: **フォームから**
   - イベントの種類: **フォーム送信時**
3. 保存 → 権限承認

### 5.6 動作確認

1. フォームをプレビューから 1 件テスト送信
2. スクリプトエディタ → **実行数**（履歴）で `onFormSubmit` が成功していることを確認
3. スプレッドシートを開いて、最下行に回答が追記されていることを確認
4. Q10 で写真を添付した場合は、`プロフィール写真` 列に Drive URL が入っていることを確認（URL クリックで画像が開ける）
5. 確認後、テスト行は削除する

## 6. 運用

### 6.1 日常運用

1. フォーム URL をゼミ生に配布（フォーム編集画面 → 右上「送信」→ リンクタブ）
2. 送信されるたびにスプレッドシートに 1 行追記される
3. 管理者はスプレッドシートを開いて内容を確認
4. 不備があれば送信者に修正依頼 / 問題なければそのまま保管

### 6.2 列を追加・変更する

フォームの質問タイトルを変更したり項目を追加した場合:

1. [docs/gas/member-form.gs](./gas/member-form.gs) の `HEADERS` 配列を新しいタイトルに合わせて編集
2. スプレッドシートの 1 行目（既存ヘッダ）を手で更新するか、**既存シートを削除** すれば GAS が次回送信時に新しいヘッダで作り直す

### 6.3 写真の扱い

- フォームに添付された画像は **フォームの設定に紐付いた Drive フォルダ** に自動保存される（フォーム編集画面 → 「質問」タブ → 「プロフィール写真」項目の設定に Drive フォルダが表示される）
- GAS はその画像の Drive URL をスプレッドシートに書くだけ
- 画像本体を他の場所にコピーしたい場合は、Drive で手動で移動 / コピー

### 6.4 重複送信

現状は毎回新規行として追記する。同じ人が複数回送信した場合は、スプレッドシート側で手動で古い行を削除する。

## 7. トラブルシューティング

| 症状 | 原因と対処 |
|---|---|
| `onFormSubmit` 実行時に `Cannot read property 'response' of undefined` | トリガーが「スプレッドシートから」になっている可能性。**フォームから / フォーム送信時** に設定し直す（§5.5）。 |
| `スクリプトプロパティ SPREADSHEET_ID が未設定です` | §5.4 の手順でスクリプトプロパティを登録できていない。キー名の typo（`SPREADSHEETID` 等）にも注意。 |
| `Exception: You do not have permission to call SpreadsheetApp.openById` | スプレッドシートの共有設定で、スクリプト実行者が閲覧 / 編集できる状態になっていない。スプレッドシートを GAS 実行ユーザーと共有する。 |
| スプレッドシートにヘッダが書き込まれているが値が入らない列がある | フォームの質問タイトルと `HEADERS` 配列の文字列が不一致。カッコ（全角 `（` / 半角 `(`）や空白の差異に注意。完全一致が必要。 |
| 写真列が空 / 「id=... 取得失敗」と出る | GAS 実行ユーザーが該当 Drive ファイルにアクセスできていない。通常はフォーム所有者＝スクリプト所有者で発生しないが、共同編集フォームで別ユーザーが送信した場合に起こり得る。 |
| 送信のたびにヘッダが何度も書かれる | `getLastRow() === 0` の判定が効いていない。シートに何らかの初期行が入っている可能性。シートを空（全行削除）にしてから再送信する。 |
