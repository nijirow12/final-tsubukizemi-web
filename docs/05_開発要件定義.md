# 開発要件定義

## 基本方針
- **コスト最小化**: 使用するサービス・ツールはすべて無料枠で運用する
- **CMS駆動**: メンバー管理・コンテンツ更新はmicroCMSで非エンジニアでも操作可能に
- **Instagram連携**: Behold.so経由でInstagram投稿を取得して表示（Graph API直接利用は廃止）

## 2. 画像管理要件

### 2.1 画像ファイルの国名タグ（必須）

地球儀のピンと画像を対応させるため、すべての活動写真に**国名タグ**が必須。

#### 管理方法: microCMS で一元管理

画像をファイルシステムではなく microCMS の「活動写真」APIで管理し、国名タグをフィールドとして持たせる。

```
API: activity-photos（活動写真）
─────────────────────────────────────
フィールド:
  - image        : 画像（メディア）      ← 写真ファイル
  - country      : 国名コード（セレクト） ← "cambodia", "vietnam", "thailand" 等
  - countryLabel : 国名表示（テキスト）   ← "カンボジア", "ベトナム" 等
  - caption      : 説明文（テキスト）     ← 活動の概要
  - date         : 撮影日（日付）
  - pinId        : 対応ピンID（テキスト） ← 地球儀ピンとの紐付け
```

#### ピンとの対応

```typescript
// ピンデータ（コード内で定義 or microCMS）
type GlobePin = {
  id: string            // "cambodia", "vietnam" 等
  lat: number
  lng: number
  label: string         // "Cambodia"
  labelJa: string       // "カンボジア"
}

// microCMS から取得した写真を country でフィルタし、ピンに紐付け
// 例: country === "cambodia" の写真 → カンボジアピンのパネルに表示
```

#### なぜ microCMS で管理するか
- ファイルシステムに画像を置く場合、タグ情報の管理が煩雑
- microCMS なら管理画面から画像アップロード + 国名選択が1画面で完結
- 写真の追加・削除・並び替えがノーコードで可能

## 3. microCMS 設計

### 3.1 API 構成

microCMS Hobbyプランでは **API 3つまで** 無料。以下の3つで構成する。

| API名 | 型 | 用途 |
|--------|------|------|
| `members` | リスト | メンバー管理 |
| `news` | リスト | ニュース記事 |
| `activity-photos` | リスト | 活動写真（国名タグ付き） |

### 3.2 members API（メンバー管理）

メンバーの追加・削除・自己紹介の編集をすべてCMSで完結させる。

```
API: members（メンバー）
─────────────────────────────────────
フィールド:
  - name         : 名前（テキスト）
  - nameEn       : 名前英語（テキスト）
  - role         : 役割（セレクト）       ← "professor", "student"
  - grade        : 学年（テキスト）       ← "4年", "3年" 等
  - portrait     : プロフィール写真（メディア）
  - introduction : 自己紹介（リッチエディタ）
  - fields       : 専門・関心領域（複数値テキスト）
  - socialLinks  : SNSリンク（繰り返し）
  - order        : 表示順（数値）
  - isActive     : 在籍中フラグ（真偽値） ← 卒業生の非表示切り替え
```

**運用フロー**:
- 新メンバー追加 → microCMS管理画面で「新規作成」→ 写真・自己紹介を入力 → 公開
- メンバー卒業 → `isActive` を false に → サイトから非表示
- 自己紹介更新 → microCMS管理画面で該当メンバーを編集 → 即時反映

### 3.3 news API（ニュース）

```
API: news（ニュース）
─────────────────────────────────────
フィールド:
  - title       : タイトル（テキスト）
  - body        : 本文（リッチエディタ）
  - thumbnail   : サムネイル（メディア）
  - category    : カテゴリ（セレクト）    ← "活動報告", "お知らせ", "メディア掲載" 等
  - publishedAt : 公開日（日付）
```

### 3.4 activity-photos API（活動写真）

前述の「2.1 画像ファイルの国名タグ」参照。

### 3.5 microCMS 連携実装

```typescript
// src/lib/microcms.ts
import { createClient } from 'microcms-js-sdk'

export const client = createClient({
  serviceDomain: process.env.MICROCMS_SERVICE_DOMAIN!,
  apiKey: process.env.MICROCMS_API_KEY!,
})

// メンバー取得（在籍中のみ、表示順でソート）
export const getMembers = () =>
  client.getList({
    endpoint: 'members',
    queries: {
      filters: 'isActive[equals]true',
      orders: 'order',
    },
  })

// 活動写真を国名でフィルタ
export const getPhotosByCountry = (country: string) =>
  client.getList({
    endpoint: 'activity-photos',
    queries: {
      filters: `country[equals]${country}`,
      orders: '-date',
    },
  })
```

## 4. Instagram 連携（Behold.so）

### 4.1 概要

地球儀のピンクリック時に、Instagram投稿から取得した**活動写真**を表示する。
Instagram Graph API の直接利用ではなく、**Behold.so** をプロキシとして使用する。

### 4.2 なぜ Behold.so を使うか（Instagram Graph API との比較）

| 項目 | Behold.so | Instagram Graph API 直接 |
|------|-----------|------------------------|
| トークン管理 | **不要（Beholdが内部管理）** | 60日ごとのリフレッシュ必須 |
| セットアップ | 管理画面で数クリック | Meta App作成・権限設定が必要 |
| ハッシュタグ | **`hashtags` 配列でパース済み** | captionから自前パースが必要 |
| 画像最適化 | **4サイズのWebP自動生成** | 元画像のみ |
| APIキー/シークレット | **不要（feedIdのみ）** | サーバーサイド管理必須 |
| クライアント呼び出し | **安全に可能** | トークン露出リスク |
| コスト | 無料（6投稿、1,200ビュー/月） | 無料（200リクエスト/時） |

### 4.3 前提条件

| 項目 | 要件 |
|------|------|
| Instagramアカウント | **Business** または **Creator** アカウント（必須） |
| Behold.soアカウント | 無料（Freeプラン） |
| Reactパッケージ | `@behold/react`（インストール済み） |
| コスト | 無料 |

### 4.4 セットアップ手順

```
1. ゼミのInstagramアカウントをBusinessアカウントに切り替え
   └→ 設定 > アカウント > プロフェッショナルアカウントに切り替え

2. Behold.so でアカウント作成（無料）
   └→ https://behold.so

3. Sources → 「+ Add source」でInstagramアカウントを認証・接続

4. Feeds → 「+ Add feed」でJSON Feedを作成
   └→ 接続したアカウントを選択
   └→ Output type: 「JSON Feed」

5. 作成されたフィードのfeedIdをコピー
   └→ Endpoint URL: https://feeds.behold.so/{feedId}

6. .env.local に設定
   └→ NEXT_PUBLIC_BEHOLD_FEED_ID={feedId}
```

### 4.5 Behold.so のデータ構造

Behold.so の JSON Feed エンドポイントが返すデータ:

```typescript
type BeholdPost = {
  id: string
  timestamp: string          // ISO 8601
  permalink: string          // Instagram投稿URL
  mediaType: 'IMAGE' | 'VIDEO' | 'CAROUSEL_ALBUM'
  mediaUrl: string           // 元画像URL
  caption: string            // 元キャプション
  prunedCaption: string      // ハッシュタグ除去済み
  hashtags: string[]         // ← パース済み配列！自前パース不要
  sizes: {                   // ← 最適化WebP自動生成！
    small:  { mediaUrl, width, height }  // 400px
    medium: { mediaUrl, width, height }  // 700px
    large:  { mediaUrl, width, height }  // 1000px
    full:   { mediaUrl, width, height }  // 2000px
  }
  likeCount: number
  commentsCount: number
}
```

### 4.6 トークン管理

**Behold.so を使うため、トークン管理は不要。**

- Behold が内部で Instagram Graph API のトークンを管理
- 開発者がトークンのリフレッシュや失効を心配する必要なし
- Cron Job も不要
- パスワード変更時も、Behold の管理画面で再接続するだけ

## 5. Instagram投稿 × 地球儀ピン連携

### 5.1 方針: ハッシュタグ条件一致によるピン座標の特定

Behold.so は投稿のハッシュタグを**パース済みの配列**で返すため、
配列の各要素をピン座標マスタの国名と照合するだけで紐付けが完了する。

#### 処理フロー

```
  ┌──────────────────────┐
  │  Behold.so JSON Feed │
  │  投稿一覧を取得       │
  └────────┬─────────────┘
           │
           ▼
  ┌──────────────────────┐
  │ 各投稿の hashtags     │
  │ 配列をそのまま使用     │
  │                      │
  │ hashtags:            │
  │   ["カンボジア",       │
  │    "tsubukizemi"]    │
  └────────┬─────────────┘
           │
           ▼
  ┌────────────────────┐     ┌──────────────────────────────┐
  │ hashtags 配列を照合 │     │ ピン座標マスタ + 日本語マップ   │
  │                    │     │                              │
  │ "カンボジア"        │────→│ カンボジア → cambodia         │
  │ "tsubukizemi"      │     │ cambodia → (11.55, 104.92)   │
  │                    │ 条件 │                              │
  │                    │ 一致 │ ベトナム → vietnam            │
  │                    │     │ vietnam → (14.05, 108.27)    │
  └────────────────────┘     └──────────────────────────────┘
           │
           ▼
  ┌────────────────────┐
  │ 一致した国のピンに   │
  │ 投稿を紐付け        │
  └────────────────────┘
```

#### 日本語・英語どちらのハッシュタグにも対応

```typescript
// src/lib/behold.ts

// 日本語ハッシュタグ → CountryId のマッピング
const JA_TAG_MAP: Record<string, CountryId> = {
  'カンボジア': 'cambodia',
  'ベトナム': 'vietnam',
  'タイ': 'thailand',
  'マレーシア': 'malaysia',
  // ...
}

// hashtags配列から国名を特定
export function extractCountryFromHashtags(hashtags: string[]): CountryId | null {
  for (const tag of hashtags) {
    if (COUNTRY_KEYS.includes(tag.toLowerCase() as CountryId)) return tag.toLowerCase() as CountryId
    if (tag in JA_TAG_MAP) return JA_TAG_MAP[tag]
  }
  return null
}
```

#### 運用ルール

Instagram投稿時に、**国名のハッシュタグ（日本語 or 英語）を含める**:

```
  #カンボジア or #cambodia   → カンボジアのピンに対応
  #ベトナム or #vietnam      → ベトナムのピンに対応
  #タイ or #thailand         → タイのピンに対応
  #マレーシア or #malaysia   → マレーシアのピンに対応
  ...等
```

## 6. 環境変数一覧

```env
# microCMS
MICROCMS_SERVICE_DOMAIN=xxxx
MICROCMS_API_KEY=xxxx

# Behold.so（Instagram Feed）
NEXT_PUBLIC_BEHOLD_FEED_ID=xxxx     # Behold フィードID（クライアントから安全に使用可能）
```

- `NEXT_PUBLIC_` プレフィックス付きで**クライアントから安全に呼べる**（feedIdにはシークレットが含まれない）
- `.env.local` は `.gitignore` に必ず含める
- 本番環境では Vercel の管理画面 > Settings > Environment Variables で設定
