# 開発要件定義

**最終更新:** 2026-04-13（実装完了後の現状反映版）
**ドキュメントの位置づけ:** 本書は、実装完了後のコードベースをベースに、設計思想と実装を整合させた「現状スナップショット」です。今後の改修・運用時にはこの文書を正とします。

## 基本方針
- **コスト最小化**: 使用するサービス・ツールはすべて無料枠で運用する
- **運用の分離**: エンジニアでなくても更新できる領域と、コード変更を伴う領域を明確に分ける
- **二系統の画像ソース**: 活動写真は「Google Drive（ゼミ所有フォルダ）」と「Instagram（Behold.so経由）」の二系統から取得し、地球儀ピン・活動ページで使い分ける
- **メンバー管理は microCMS 連携を前提に設計済み**（現状はデモデータ運用、切替は1ファイルで可能）

## 2. 画像管理要件（実装済み構成）

本サイトの活動写真は、**Google Drive**（ゼミ所有フォルダの階層構造）と **Instagram**（Behold.so経由）の二系統から取得する。国名タグは microCMS のフィールドではなく、**Google Drive のフォルダ名**および **Instagram 投稿のハッシュタグ** で管理する。

### 2.1 なぜ microCMS `activity-photos` API を採用しなかったか（設計変更の経緯）

初期設計では `activity-photos` API を microCMS に設置する予定だったが、以下の理由で **Google Drive + Instagram 二系統** に変更した。

| 観点 | microCMS `activity-photos` | Google Drive + Instagram（採用） |
|------|---------------------------|-----------------------------|
| 写真の量 | Hobbyプランの容量制限に抵触しやすい | Drive は実質無制限、Instagram は Behold 経由 |
| 非エンジニア運用 | 管理画面から1枚ずつアップロード必須 | **Drive へドラッグ&ドロップで完了** |
| 現場での運用 | 渡航先でCMSログインが必要 | **Instagram に普段どおり投稿するだけで反映** |
| 国別タグ付け | CMS フィールドとして保持 | **フォルダ名 or ハッシュタグで自動判定** |
| バックアップ | microCMS 単一障害点 | Drive と Instagram に二重化 |

運用担当者が「渡航先で Instagram に投稿する」「帰国後に未投稿の写真を Drive にまとめて置く」という日常的な行動だけで完結することを重視し、この構成を採用した。

### 2.2 Google Drive ソース（地球儀ピン用）

**役割:** 3D地球儀のピンに紐づく写真パネルを表示する。

**フォルダ構造（厳守）:**

```
ルートフォルダ (GOOGLE_DRIVE_ROOT_FOLDER_ID で指定)
├── Asia/                    ← 第1階層: 地域（任意の名前でOK）
│   ├── 🇰🇭カンボジア/        ← 第2階層: 国名（CountryId に自動マッチ）
│   │   ├── img001.jpg
│   │   └── img002.jpg
│   ├── 🇻🇳ベトナム/
│   ├── 🇹🇭タイ/
│   └── ...
└── ...
```

**国名判定ロジック:** [src/lib/google-drive.ts](../src/lib/google-drive.ts) の `matchFolderToCountry()` が、フォルダ名に **日本語国名（例: `カンボジア`）** または **英語国名（例: `cambodia`）** が含まれているかをマッチングし、`CountryId` を決定する。絵文字や接頭辞（🇰🇭 など）は無視される。長い国名優先（`インドネシア` が `インド` より優先）。

**API エンドポイント:** 以下の3つの Next.js API Route 経由で Drive にアクセスする。クライアント側からは直接 Google Drive を叩かない（サービスアカウントの秘匿のため）。

| エンドポイント | 役割 |
|--------------|------|
| `GET /api/drive/folders` | ルート配下の国フォルダを2階層探索してフラットに返す |
| `GET /api/drive/images?folderId=...` | 指定フォルダ内の画像一覧を取得（最大50件、作成日降順） |
| `GET /api/drive/image?fileId=...` | 画像本体をサムネイル化（`=s600`）して JPEG 配信、`Cache-Control: public, max-age=86400` |

**画像処理:**
- すべての画像は Google Drive の `thumbnailLink` 経由で **最大600px の JPEG に自動圧縮**（HEIC 対応）
- 原画がない場合のみ `alt=media` で原画をフォールバック配信
- クライアント側で24時間キャッシュ

**認証:**
- Google Cloud のサービスアカウント経由
- 環境変数 `GOOGLE_SERVICE_ACCOUNT_JSON` に JSON 文字列を格納（フォールバックで `tsubukizemi-demo-*.json` のキーファイルも読める。開発時のみ）
- スコープ: `drive.readonly`
- ルートフォルダにサービスアカウントのメールアドレスを **閲覧者権限で共有** する必要あり

### 2.3 Instagram ソース（Behold.so 経由、活動ページ用）

**役割:** `/activities` ページに最新の Instagram 投稿を表示する。地球儀ピンの補助ソースとしても動作可能な実装。

**前提:**
- Instagram アカウントは **Business** または **Creator** アカウント（必須）
- Behold.so で Instagram アカウントを接続し、JSON Feed を発行
- 環境変数 `NEXT_PUBLIC_BEHOLD_FEED_ID` に Feed ID を設定
- Feed URL: `https://feeds.behold.so/{FEED_ID}`（クライアントから直接取得可能。feedId にシークレット含まれず安全）

**Behold.so のデータ構造:**

```typescript
type BeholdPost = {
  id: string
  timestamp: string
  permalink: string
  mediaType: 'IMAGE' | 'VIDEO' | 'CAROUSEL_ALBUM'
  mediaUrl: string
  thumbnailUrl?: string
  caption: string
  prunedCaption: string
  hashtags: string[]         // ← Behold が自動パース
  mentions: string[]
  likeCount: number
  commentsCount: number
  sizes: {
    small:  { mediaUrl, width, height }  // ~400px
    medium: { mediaUrl, width, height }  // ~700px
    large:  { mediaUrl, width, height }  // ~1000px
    full:   { mediaUrl, width, height }  // ~2000px
  }
  children?: BeholdChild[]   // CAROUSEL_ALBUM の複数画像
}
```

**ハッシュタグによる国判定:** [src/lib/behold.ts](../src/lib/behold.ts) の `extractCountryFromHashtags()` が、投稿の `hashtags` 配列から日本語タグ（`カンボジア`等）または英語タグ（`cambodia`等）を照合し、`CountryId` を返す。

**カルーセル対応:** `mediaType === 'CAROUSEL_ALBUM'` の場合は `children` を展開して全画像を取り出す。動画はサムネイル（`thumbnailUrl`）で代替。

## 3. microCMS 設計（現状: 未活性、設計済み）

### 3.1 現状の実装状態

**現時点で有効な API:** なし（microCMS は接続されていない、または APIキーは発行済みだがコード側で未使用）

**コード側の準備:**
- [src/lib/microcms.ts](../src/lib/microcms.ts) — `getMembers()` 関数実装済み
- [src/app/api/members/route.ts](../src/app/api/members/route.ts) — microCMS 連携コードがコメントアウト済みで、現在は `src/data/members-demo.ts` のハードコードを返す

**CMS 連携を有効化するには:**
[src/app/api/members/route.ts](../src/app/api/members/route.ts) のコメントアウトを解除し、import を差し替えるだけ。

```typescript
// Before（現状）
import { demoMembers } from '@/data/members-demo'
export async function GET() {
  return NextResponse.json(demoMembers)
}

// After（CMS 連携時）
import { getMembers } from '@/lib/microcms'
export async function GET() {
  try {
    const members = await getMembers()
    return NextResponse.json(members)
  } catch (err) {
    console.error('microCMS fetch error:', err)
    return NextResponse.json([], { status: 500 })
  }
}
```

### 3.2 members API スキーマ（現行の `microcms.ts` と整合）

[src/lib/microcms.ts](../src/lib/microcms.ts) で定義されている `Member` 型に合わせて、microCMS 側の API を以下のスキーマで作成する。

```
API: members（メンバー）
エンドポイントID: members
型: リスト形式
─────────────────────────────────────
フィールド:
  - fullName : テキスト（必須）        ← "山田 太郎"
  - photo    : 画像（メディア）          ← プロフィール写真
  - role     : テキスト                  ← "4年" / "3年" / "OB" 等
  - snsUrl   : テキスト                  ← Instagram / LinkedIn 等のURL
```

> **注意:** 現行のデモデータ (`src/data/members-demo.ts`) はもっと多くのフィールド（`nameEn`, `introduction`, `introductionEn`, `projects`, `projectsEn`, `countries`, `isOB`, `grade`）を持つ。CMS へ完全移行する場合は、`src/lib/microcms.ts` の `Member` 型と microCMS スキーマを**デモデータに合わせて拡張**することを推奨する。

## 4. Instagram 連携（Behold.so）

### 4.1 概要

`/activities` ページで Instagram 投稿を一覧表示する。また地球儀ピンの補助的な画像ソースとしても利用可能。

### 4.2 なぜ Behold.so を使うか

| 項目 | Behold.so | Instagram Graph API 直接 |
|------|-----------|------------------------|
| トークン管理 | **不要（Behold 側で管理）** | 60日ごとのリフレッシュ必須 |
| セットアップ | 管理画面で数クリック | Meta App 作成・権限設定が必要 |
| ハッシュタグ | **`hashtags` 配列でパース済み** | caption から自前パース |
| 画像最適化 | **4サイズ自動生成** | 元画像のみ |
| シークレット | **feedId のみ（公開OK）** | トークン管理必須 |
| クライアント呼び出し | **安全に可能** | トークン露出リスク |
| コスト | 無料（6投稿、1,200ビュー/月） | 無料（200リクエスト/時） |

### 4.3 セットアップ手順

```
1. ゼミの Instagram アカウントを Business アカウントに切り替え
   └→ 設定 > アカウント > プロフェッショナルアカウントに切り替え

2. Behold.so でアカウント作成（無料）
   └→ https://behold.so

3. Sources → 「+ Add source」で Instagram を接続

4. Feeds → 「+ Add feed」で JSON Feed を作成

5. feedId をコピー
   └→ Endpoint URL: https://feeds.behold.so/{feedId}

6. Vercel 環境変数に登録
   └→ NEXT_PUBLIC_BEHOLD_FEED_ID={feedId}
```

### 4.4 トークン管理

Behold が内部で Instagram Graph API のトークンを管理するため、**開発者側でのトークン更新作業は不要**。Cron Job も不要。Instagram のパスワードを変更した際のみ、Behold 管理画面で再接続すればよい。

## 5. Instagram投稿 × 地球儀ピン連携

### 5.1 方針: ハッシュタグと国コードの照合

Behold.so は投稿のハッシュタグを**パース済み配列**で返すため、配列の各要素を [src/data/pin-coordinates.ts](../src/data/pin-coordinates.ts) の `CountryId` と照合するだけで紐付けが完了する。

#### 処理フロー

```
  ┌──────────────────────┐
  │  Behold.so JSON Feed │
  │  投稿一覧を取得       │
  └────────┬─────────────┘
           │
           ▼
  ┌──────────────────────┐
  │ 各投稿の hashtags     │
  │ 配列をそのまま使用     │
  │                      │
  │ hashtags:            │
  │   ["カンボジア",       │
  │    "tsubukizemi"]    │
  └────────┬─────────────┘
           │
           ▼
  ┌────────────────────┐     ┌──────────────────────────────┐
  │ hashtags 配列を照合 │     │ ピン座標マスタ + 日本語マップ   │
  │                    │     │                              │
  │ "カンボジア"        │────→│ カンボジア → cambodia         │
  │ "tsubukizemi"      │     │ cambodia → (11.55, 104.92)   │
  │                    │ 条件 │                              │
  │                    │ 一致 │ ベトナム → vietnam            │
  │                    │     │ vietnam → (14.05, 108.27)    │
  └────────────────────┘     └──────────────────────────────┘
           │
           ▼
  ┌────────────────────┐
  │ 一致した国のピンに   │
  │ 投稿を紐付け        │
  └────────────────────┘
```

## 7. 環境変数一覧

```env
# --- microCMS（メンバー管理。現状は未活性だがコード側は準備済み） ---
MICROCMS_SERVICE_DOMAIN=xxxx
MICROCMS_API_KEY=xxxx

# --- Google Drive（活動写真） ---
GOOGLE_DRIVE_ROOT_FOLDER_ID=xxxx        # ルートフォルダのID
GOOGLE_SERVICE_ACCOUNT_JSON={"type":"service_account",...}  # JSONの中身を文字列で

# --- Behold.so（Instagram Feed） ---
NEXT_PUBLIC_BEHOLD_FEED_ID=xxxx         # クライアントから安全に利用可能
```

**取り扱いルール:**
- `NEXT_PUBLIC_` プレフィックスが付いた変数のみブラウザに露出する。それ以外は絶対に付けない。
- `.env.local` は `.gitignore` に含める（`.gitignore` で `.env*.local` を除外済み）。
- 本番環境では Vercel ダッシュボード → Settings → Environment Variables で設定する。
- Google サービスアカウント JSON は機密情報。`tsubukizemi-demo-*.json` というファイル名パターンで `.gitignore` されている。
